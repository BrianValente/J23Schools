Capítulo 2: Sintaxis

JS puede ser implementado con las etiquetas <script>…</script> en el código HTML. Pueden ponerse en cualquier parte del código de la página, pero lo mejor es ponerlo entre las etiquetas <head> y </head>.

Las etiquetas <script> avisan al navegador que todo lo que esté luego de las mismas es un script de JS.

Una sintaxis correcta luce de la siguiente manera:

  <head>    Head code    <script language=“javascript” type=“text/javascript”>      Script JS    </script>   </head>

Donde los atributos de la etiqueta script significa:

  • Language: qué lenguaje utiliza el script. Normalmente se obvia este campo, ya que es sabido que se tratará de JS. Las últimas versiones de HTML y XHTML no reconocen ya este atributo.
  • Type: indica qué tipo de lenguaje se utiliza.


Creando el primer código

Vamos a hacer el ejemplo más básico de todos: mostrar en pantalla el clásico “¡Hola Mundo!”.

Para escribir una línea de texto en la página, llamamos a la función document.write. Esta función sirve para colocar texto, modificar una parte del HTML o ambas. Escribí el ejemplo en la consola de al lado y mirá lo que pasa.

 <html>   <body>    <script language="javascript" type="text/javascript">     <!--     document.write("Hello World!")     //-->    </script>   </body>  </html>

Prácticas de JS

JS ignora los espacios, tabulaciones y enters que estén en los programas de JavaScript. Podés usarlos para formatear e identificar tu código, para hacerlo más fácil de leer e interpretar, pero JS no va a notar la diferencia si está ahí o no.

Punto y coma: Cada línea de código finaliza con un punto y coma, al igual que en cualquier lenguaje (C, C++, Java…), pero JS te permite omitirlo si cada línea de código está en una línea distinta. Estos códigos son comprensibles tanto uno como el otro, pero es una buena práctica siempre poner un punto y coma.

Un código sólo tabulado por enter luce de esta manera:

 <script language="javascript" type="text/javascript">   <!--    var1 = 10    var2 = 20   //-->  </script>

Mientras que con punto y coma, puede realizarse de esta forma y ser igualmente efectivo:

 <script language="javascript" type="text/javascript">   <!--    var1 = 10; var2 = 20;   //-->  </script>

Sensibilidad ante las mayúsculas (Case Sensitivity): JS es un lenguaje que distingue entre mayúsculas y minúsculas, lo cual significa que las variables, los nombres de las funciones, y cualquier otra identificación deben ser siempre escritos con la misma capitalización de las letras.

Es decir: una variable que se llame hola, no es lo mismo que se una que se llame Hola ni HOLA. Para JS, serán tres variables diferentes.


Comentarios en JavaScript: JavaScript identifica comentarios del estilo de C, C++ y HTML. Esto significa que:

  • Cualquier texto con // al principio es calificado como un comentario de una sola línea.
  • Cualquier texto entre /* y */ es un comentario, y puede ocupar múltiples líneas.
  • Cualquier texto entre <!-- y --> es interpretado como comentario de una sola línea.

El siguiente ejemplo muestra cómo usar comentarios en JS:

 <script language="javascript" type="text/javascript">   <!--    // This is a comment. It is similar to comments in C++    /*    * This is a multiline comment in JavaScript    * It is very similar to comments in C Programming    */   //-->  </script>
