Capítulo 1: Un poco de teoría

¿Qué es JavaScript (JS)?

JS es un lenguaje de programación dinámico, ligero y utilizado mayormente como parte de las páginas web: su implementación permite que el cliente (es decir, vos como usuario) puedas interactuar con la página.

Antes, JS se conocía como LiveScript de Netscape, pero cambió su nombre a JS posiblemente por la emoción que generó Java en su lanzamiento. Apareció por primera vez en 1995, implementado en Netscape, Internet Explorer, entre otros navegadores.


JS del lado del Cliente

JS del lado del cliente es la forma más común de JavaScript. Es decir, el script está incluido o referido en un documento HTML para que su código sea interpretado por el navegador.

Esto significa que una página web no es sólo un código HTML estático, si no que también incluye programas que interactúan con el usuario, controlan el navegador y hacen y deshacen el código HTML según diga el código.


JS del lado del cliente provee muchas ventajas con respecto a los tradicionales scripts del lado del servidor, por ejemplo: puede chequear que el usuario haya ingresado un e-mail válido en la casilla del formulario. ¿Cómo hace esto? El código JS se ejecuta al enviar el formulario, y tras revisar que todas las casillas estén correctamente completadas, es entonces cuando la envían al servidor web.

Ventajas:

  • Menos interacción del servidor: los datos pueden ser chequeados antes de enviarse al servidor, lo que resulta en menos tráfico.
  • Feedback inmediato a los visitantes: los usuarios no tienen que esperar a que la página se recargue para ver si ingresaron todo.
  • Interacción incrementada: las interfaces pueden reaccionar cuando el usuario las toca con el mouse o presiona cierta tecla del teclado.
  • Interfaces más atractivas: se pueden agregar componentes fuera de los que HTML o CSS pueden proporcionar, y llamar más la atención de esta manera.

Desventajas:

  • La codificación de JS del lado del cliente no permite la lectura ni la escritura de archivos, mayormente por cuestiones de seguridad
  • No tiene soporte para aplicaciones de red
  • JS no puede manipular múltiples procesadores

Su única funcionalidad es, como se ha dicho varias veces ya, el crear interacción entre el usuario y la página web.


¿Y con qué codifico?

Realmente, JS no necesita nada del otro mundo. Simplemente con utilizar un editor de texto como Bloc de notas es suficiente. Te recomendamos el uso de Atom o Sublime Text, que son gratuitos y muy personalizables.
