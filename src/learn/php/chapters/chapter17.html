Variables globales

Varias variables predefinidas en PHP son "superglobals", lo que significa que siempre son accesibles, independientemente del alcance - y se puede acceder a ellos desde cualquier función, clase o archivo sin tener que hacer nada especial. El variables de PHP superglobal son:

  • $ GLOBALS
  • $ _SERVER
  • $ _REQUEST
  • $ _POST
  • $ _GET
  • $ _FILES
  • $ _ENV
  • $ _COOKIE
  • $ _SESSION

$ GLOBALS

$ GLOBALS es una variable súper mundial PHP que se utiliza para acceder a las variables globales desde cualquier parte del script PHP (también desde el interior de funciones o métodos). Tiendas PHP todas las variables globales en una matriz llamada $ GLOBALS [ índice ]. El índice contiene el nombre de la variable. El siguiente ejemplo muestra cómo utilizar el súper variable global $ GLOBALS:

<?php $x = 75; $y = 25; function addition() { $GLOBALS['z'] = $GLOBALS['x'] + $GLOBALS['y']; } addition(); echo $z; ?>

$ _SERVER

$ _SERVER Es una variable súper mundial PHP que contiene información acerca de los encabezados, rutas y ubicaciones de scripts. El siguiente ejemplo muestra cómo utilizar algunos de los elementos en $ _SERVER:

<?php echo $_SERVER['PHP_SELF']; echo "<br>"; echo $_SERVER['SERVER_NAME']; echo "<br>"; echo $_SERVER['HTTP_HOST']; echo "<br>"; echo $_SERVER['HTTP_REFERER']; echo "<br>"; echo $_SERVER['HTTP_USER_AGENT']; echo "<br>"; echo $_SERVER['SCRIPT_NAME']; ?>

Código Descripción
$_SERVER['PHP_SELF'] Devuelve el nombre de el script ejecutado
$_SERVER['GATEWAY_INTERFACE'] Devuelve la versión del Common Gateway Interface (CGI) que está usando el servidor
$_SERVER['SERVER_ADDR'] Devuelve la IP del servidor
$_SERVER['SERVER_NAME'] Devuelve el nombre del servidor
$_SERVER['SERVER_SOFTWARE'] Devuelve la indentificación del server
$_SERVER['SERVER_PROTOCOL'] Devuelve el nombre y revisión del protocolo
$_SERVER['REQUEST_METHOD'] Devuelve el metodo requerido para acceder a la página
$_SERVER['REQUEST_TIME'] Devuelve el tiempo que tardó en entrar a la página
$_SERVER['QUERY_STRING'] Devuelve la cadena de consulta si la página es accedida mediante una cadena de consulta/td>
$_SERVER['HTTP_ACCEPT'] Devuelve la aceptación del header de la consulta
$_SERVER['HTTP_ACCEPT_CHARSET'] Devuelve el Accept_Charset header de la consulta
$_SERVER['HTTP_HOST'] Devuelve el host header de la consulta
$_SERVER['HTTP_REFERER'] Devuelve el url completo
$_SERVER['HTTPS'] Devuelve el script consultado mediante un protocolo HTTP securo
$_SERVER['REMOTE_ADDR'] Devuelve la IP de donde el usuario está viendo
$_SERVER['REMOTE_HOST'] Devuelve el nombre del host de donde está viendo el usuario
$_SERVER['REMOTE_PORT'] Devuelve el puerto que está siendo usado para que el usuario se enlace con la página
$_SERVER['SCRIPT_FILENAME'] Devuelve la ruta absoluta del script que se está ejecutando
$_SERVER['SERVER_ADMIN'] Devuelve el valor dado a la directiva SERVER_ADMIN en el archivo de configuración del servidor web
$_SERVER['SERVER_PORT'] Devuelve el puerto del servidor utilizado por el servidor web para comunicación
$_SERVER['SERVER_SIGNATURE'] Devuelve la versión del servidor y el nombre del host virtual que se agregan a páginas generadas por el servidor
$_SERVER['PATH_TRANSLATED'] Devuelve la ruta del sistema de archivos a la secuencia de comandos actual
$_SERVER['SCRIPT_NAME'] Devuelve la ruta del script actual
$_SERVER['SCRIPT_URI'] Devuelve el URI de la página actual

$ _REQUEST

$ _REQUEST se utiliza para recoger datos después de la presentación de un formulario HTML. El siguiente ejemplo muestra un formulario con un campo de entrada y un botón de envío. Cuando un usuario envía los datos haciendo clic en "Enviar", los datos del formulario se envía al archivo especificado en el atributo action de la etiqueta <form>. En este ejemplo, nosotros señalamos el propio archivo de datos de formularios de procesamiento. Si desea utilizar otro archivo PHP para procesar los datos del formulario, reemplace que con el nombre de archivo de su elección. Entonces, podemos utilizar el súper variable global $ _REQUEST para recoger el valor del campo de entrada:

<html> <body> <form method="post" action=""> Name: <input type="text" name="fname"> <input type="submit"> </form> <?php if ($_SERVER["REQUEST_METHOD"] == "POST") { // collect value of input field $name = $_REQUEST['fname']; if (empty($name)) { echo "Name is empty"; } else { echo $name; } } ?> </body> </html>

$ _POST

PHP $ _POST es ampliamente utilizado para recopilar datos de formularios después de la presentación de un formulario HTML con method = "post". $ _POST También se usa ampliamente para pasar variables. El siguiente ejemplo muestra un formulario con un campo de entrada y un botón de envío. Cuando un usuario envía los datos haciendo clic en "Enviar", los datos del formulario se envía al archivo especificado en el atributo action de la etiqueta <form>. En este ejemplo, señalamos que el propio archivo de datos de formularios de procesamiento. Si desea utilizar otro archivo PHP para procesar los datos del formulario, reemplace que con el nombre de archivo de su elección. Entonces, podemos utilizar el súper variable global $ _POST para recoger el valor del campo de entrada:

<html> <body> <form method="post" action=""> Name: <input type="text" name="fname"> <input type="submit"> </form> <?php if ($_SERVER["REQUEST_METHOD"] == "POST") { // collect value of input field $name = $_POST['fname']; if (empty($name)) { echo "Name is empty"; } else { echo $name; } } ?> </body> </html>

$ _GET

$ _GET también se puede utilizar para recopilar datos de formularios después de la presentación de un formulario HTML con method = "get". $ _GET También puede recoger datos enviados en la URL. Supongamos que tenemos una página HTML que contiene un hipervínculo con parámetros: <html> <body> <a href="test_get.php?subject=PHP&web=W3schools.com">Test $GET</a> </body> </html> Cuando un usuario hace clic en el enlace "Test $ GET", los parámetros "sujeto" y "web" son enviados a "test_get.php", y, a continuación, pueden acceder a sus valores de "test_get.php" con $ _GET. El siguiente ejemplo muestra el código en "test_get.php":

<html> <body> <?php echo "Study " . $_GET['subject'] . " at " . $_GET['web']; ?> </body> </html>
